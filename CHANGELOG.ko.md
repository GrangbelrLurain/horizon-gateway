# 변경 이력 (Changelog)

이 프로젝트의 모든 주목할 만한 변경 사항은 이 파일에 문서화됩니다.

이 형식은 [Keep a Changelog](https://keepachangelog.com/en/1.1.0/)를 기반으로 합니다.

## [v2.5.3] - 2026-07-14

### Changed (변경 사항)

- **Fluent 오버레이 스크롤바**: 메인·보조 창이 WebView2 `fluentOverlay` 스크롤바를 사용해 목록·패널에 클래식 ↑↓ 버튼 없이 스크롤됩니다.
- **테마 연동 스크롤바 폴리싱**: 앱 전역 thin pill thumb가 라이트/다크 `base-content`를 따릅니다. Chromium `scrollbar-width`는 Windows fluent 화살표를 강제해 사용하지 않습니다.

### Fixed (버그 수정)

- **Windows GUI 콘솔 창**: 릴리스 빌드에서 windows 서브시스템을 사용해 탐색기/시작 메뉴로 실행할 때 CMD 로그 창이 뜨지 않도록 했습니다. CLI 모드는 stdout/stderr가 파이프·파일로 리다이렉트되지 않은 경우에만 부모 콘솔에 연결해, 에이전트 pipe 캡처는 그대로 유지합니다.

## [v2.5.2] - 2026-07-14

### Added (추가 기능)

- **다국어 및 확장 가능한 업데이트 안내 모달**: 앱 업데이트 완료 후 최초 진입 시 무엇이 새롭게 추가되었는지 한눈에 보여주는 고급스러운 디자인의 업데이트 알림 모달을 도입했습니다.
- **상단 설정 메뉴 연동**: 언제든지 과거 업데이트 기록을 찾아볼 수 있도록 상단 바 설정 드롭다운 메뉴에 "업데이트 내역" 옵션을 신설하고 연동했습니다.
- **마크다운 동적 파싱 및 가상화 스크롤**: 컴파일 타임에 마크다운 소스(`CHANGELOG.md` / `CHANGELOG.ko.md`)를 통째로 읽어 동적으로 파싱함으로써 단일 소스로 데이터의 정합성을 보장하며, 전체 이력 조회 시에도 렉이 없는 초고속 가상화 스크롤을 구현했습니다.
- **한글 변경 이력 전체 번역본 배포**: 웹사이트와 앱 모달 모두에서 완벽한 다국어 조회가 가능하도록 프로젝트 루트에 `v1.0.0`부터의 역사를 온전히 한글로 번역한 `CHANGELOG.ko.md`를 배포했습니다.

## [v2.5.1] - 2026-07-14

### Changed (변경 사항)

- **앱 리브랜딩 마침**: `watchtower`에서 `horizon-gateway`로 남아있던 파일, 내부 변수, 디렉토리 및 셀렉터를 모두 마이그레이션했습니다.
- **안정적인 데이터 마이그레이션**: 앱을 업그레이드할 때 도메인 목록이나 설정이 유실되지 않도록, 기존의 사용자 데이터베이스(예: `domains.json`)가 존재하는지 체크하여 자동으로 설정 파일을 복사 및 마이그레이션해 주는 로직을 추가했습니다.

## [v2.5.0] - 2026-07-13

### Added (추가 기능)

- **Supabase 인증 및 데이터베이스 통합**: Supabase JS SDK를 앱에 연동했습니다. GitHub을 통한 소셜 OAuth 로그인을 지원합니다.
- **Tauri v2 딥링크 라우팅**: 앱 시작 시 Windows 레지스트리에 자동 등록되는 커스텀 URL 스킴 핸들러(`horizon-gateway://`)를 활성화했습니다.
- **Windows 단일 인스턴스 실행 및 인자 전달**: 여러 개의 창이 동시에 뜨는 것을 방지하기 위해 `tauri-plugin-single-instance`를 추가했습니다. 실행 중인 메인 창으로 OAuth 콜백 URL 등의 매개변수가 누락 없이 전달되도록 CLI 인자 수동 파싱 및 라우팅 로직을 구현했습니다.
- **GitHub Sponsors 연동**: 국내 결제 서비스 제한을 방지하기 위해 개발자 후원 플랫폼을 GitHub Sponsors로 전환했습니다. 테마에 맞게 깜빡이는 로즈(Rose) 후원 단추가 추가되었습니다.
- **얼리 액세스 연구소 및 개발자 피드백**: 후원자용 실험적 기능 설정 옵션과 Supabase DB에 직접 의견을 제출하는 개발자 피드백 폼을 사용자 설정 페이지에 추가했습니다.
- **설정 및 프로필 드롭다운 정리**: TopBar 버튼들을 재조직하여, 인프라 설정 버튼을 클릭형 설정 드롭다운 메뉴로 통합하고 프로필 관리 및 로그아웃 버튼을 하나의 깔끔한 프로필 드롭다운 메뉴로 합쳤습니다.
- **상단 바 Chevron 아이콘 제거**: 더 간결하고 일관된 레이아웃을 위해 상단 바 메뉴 단추에서 쉐브론 아이콘을 제거했습니다.
- **자동 GitHub 프로필 동기화**: OAuth 로그인을 수행할 시 사용자 메타데이터를 기반으로 프로필 이름과 아바타 이미지를 자동으로 로컬 프로필과 동기화합니다.

### Changed (변경 사항)

- **앱 리브랜딩**: 기존 `watchtower`에서 `horizon-gateway`로의 이름 및 설정 마이그레이션을 계속 진행했습니다.
- **Tauri Opener 우회**: Tauri Opener 라이브러리의 보안 제약을 우회하여 외부 브라우저 URL을 열 수 있도록 커스텀 Rust 백채널 `open_external_url` 명령을 통해 열리도록 개선했습니다.

## [v2.4.4] - 2026-07-10

### Added (추가 기능)

- **API 로그 다중 HTML 내보내기**: 여러 개의 API 로그 항목을 선택하여 하나의 통합 HTML 보고서(목차, 항목별 복사 액션, 본문 스크롤 지원 등 포함)로 내보낼 수 있는 기능이 추가되었습니다.
- **저장 및 폴더 열기**: HTML 내보내기 시 기본 저장 창(Tauri 파일 다이얼로그)을 사용하고, 성공적으로 저장 완료 시 `revealItemInDir`를 통해 "폴더 열기" 액션을 제안하도록 구현했습니다.
- **공통 다운로드 헬퍼**: 재사용 가능한 파일 다운로드 흐름을 제공하기 위해 `saveTextDownload`, `revealInFolder`, `offerRevealSavedDownload` 함수를 추가했습니다.

### Changed (변경 사항)

- **HTTP 본문 표시**: `formatHttpBody` 함수가 이스케이프 문자(`\n`, `\t`)를 원본 그대로 처리하며, 이중 인코딩된 JSON 문자열을 보기 쉽게 풀어내어 복사/추출 시 실제 데이터와 더 가깝게 가독성을 개선했습니다.
- **HTML 카드 이스케이프**: 복사 또는 내보내기용 HTML 카드 구성 시 발생 가능한 보안 이슈를 막기 위해 `<pre>` 태그 및 헤더 내 사용자 콘텐츠를 적절히 이스케이프하도록 수정했습니다.

## [v2.4.3] - 2026-07-10

### Added (추가 기능)

- **DisabledPanel (비활성 패널)**: 도메인에서 사용할 수 없는 기능 패널 진입 시, 조용히 Overview로 리다이렉트되는 대신 비활성화 상태임을 사용자에게 명시적으로 안내하는 패널을 구현했습니다. 사용자는 이 화면에서 직접 클릭하여 즉시 기능을 켤 수 있습니다.
- **도메인 주소 일괄 복사**: 새 대량 처리 툴바(`DomainListBulkToolbar`)에 선택한 모든 도메인의 URL 주소를 순서대로 클립보드에 한번에 복사하는 "URL 복사" 액션을 도입했습니다.
- **범위 선택 및 개별 토글 선택**: 도메인 대량 관리 모드에서 Shift+클릭을 통한 범위 선택과 Ctrl+클릭을 통한 개별 토글이 가능하도록 사용성을 향상하고 시각적 안내를 추가했습니다.
- **`HubSurfaceEmbedContext` 닫기 콜백**: `HubSurfaceEmbedProvider`에 `onDismiss` 프로퍼티를 추가하고 `useHubSurfaceDismiss()` 훅을 제공합니다. 이를 통해 모달 내 임베디드 오버레이(예: 도메인 추가 등)를 닫을 때 Tauri 창 전체가 닫히지 않고 내부 오버레이만 올바르게 닫힙니다.
- **다국어(i18n)**: 한글 및 영어 다국어 번들 파일에 `bulkCopyUrls`, `bulkCopied`, `bulkSelectionHint` 등의 문구를 추가했습니다.

### Changed (변경 사항)

- **도메인 변경 시 패널 깊이(Depth) 보존**: 도메인을 전환하더라도 이전에 보던 패널 깊이를 초기화하여 `overview`로 돌리는 대신 동일한 패널 상태를 유지합니다.
- **대량 관리 상태 비저장**: 사용자가 페이지를 새로고침할 시 대량 선택 상태가 남아 꼬이는 현상을 방지하기 위해 `domainListBulkModeAtom`을 일반 `atom`으로 변경했습니다.
- **대량 선택 타입 최적화**: 선택 여부를 확인하는 속도를 O(1) 수준으로 빠르게 유지하기 위해 `domainListBulkSelectedIdsAtom` 형식을 `ReadonlySet<number>`로 바꿨습니다.
- **도메인 리스트 리팩토링**: 코드 가독성과 유지보수성 향상을 위해 `DomainListPanel`에서 `DomainListItem`, `VirtualizedGroupedDomainList`, `DomainListInteractionContext`를 별도 파일로 추출했습니다.
- **`applyNavigation` 헬퍼**: URL을 네비게이션하기 전에 내부 참조값(ref)과 Jotai 상태가 완전하게 일치하도록 동기화해 주어, 비동기 레이스로 인해 두 번 페이지 이동이 일어나던 이슈를 방지합니다.

### Fixed (버그 수정)

- **도메인 선택 해제 시 참조 레이스 해결**: `selectDomain`, `clearDomain`, `restoreNavigation` 함수들이 `applyNavigation`을 사용하여 URL 네비게이션 전에 내부 참조값(`domainIdRef`/`panelsRef`)을 정확히 갱신하도록 바로잡았습니다.
- **불필요한 패널 리셋 제거**: 특정 패널에 접근 불가할 때 무조건 `overview`로 리셋하던 `useEffect` 문을 제거하고, 대신 `DisabledPanel`을 렌더링하도록 개선했습니다.
